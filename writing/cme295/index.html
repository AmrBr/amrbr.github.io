Lecture One: Transformer

NLP overview

Natural Language Processing tasks can be divided into three main categories:

1. Classification:

The input is text and the goal is to classify it into a known set of classes.

  • Language detection
  • Sentiment extraction

“This teddy bear is SO CUTE!” -> MODEL -> (+) (Positive review) Evaluation Metrics:

  • Accuracy: The percentage of observations that were correctly predicted.
  • Precision: The percentage of predicted positives that were correct.
  • Recall: The percentage of actual positives that were correctly predicted.
  • F1 score: A single metric that combines precision and recall.

Using multiple evaluation metrics is helpful when datasets are imbalanced. In that case, accuracy alone can be misleading if the model predicts only the majority class, while precision and recall show the actual performance.

2. Multi-Classification:

This is similar to classification, but instead of identifying one class, the goal is to classify multiple parts of the input text.

  • POS tagging
  • Named entity recognition

“A cute teddy bear is reading” -> MODEL -> “A cute teddy bear is reading”

The evaluation metrics are the same as for classification, but applied at the token or word level rather than the sentence level.

3. Generation:

This differs from the previous two types because the output is variable-length text and the exact length is not known in advance.

  • Machine translation
  • Question answering

“A cute teddy bear is reading” -> MODEL -> “Un ours en peluche mignon lit”

Evaluation Metrics:

  • BLEU: Measure of translation quality, similar to precision.
  • ROUGE: Measure of generation quality, similar to recall.

Tokenization

Models do not understand words; they understand numbers. To convert text into numbers, the process goes through several stages. The first stage is tokenization, and its main goal is to split text into tokens that are easier for the model to process. There are multiple approaches to tokenization.

OOV -> Out Of Vocab, which means that this word/token was not in the training vocab for that model.

Word Representation

This is the second stage, which transforms tokens from the first stage into numbers—specifically vectors in an n-dimensional space where each token is represented by a vector. The goal is to place similar tokens close together and dissimilar tokens farther apart. For example, words like eye and see should be closer than eye and pen. This closeness is measured by cosine similarity: vectors in the same direction have high similarity, while orthogonal vectors have low similarity.

The naive approach is one-hot encoding. For example, with a vocabulary of size 3, each vector has length 3 and a single index is 1 to indicate the word, while all others are 0: (1, 0, 0), (0, 1, 0), and (0, 0, 1).

The issue is that all one-hot vectors are orthogonal, so they do not capture semantic meaning. Each token is treated independently, which is why models instead use learned embeddings that are trained from data.
